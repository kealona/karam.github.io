风格指南

必要的风格

  1. 组件名应该为多个单词，可以避免以后和HTML元素冲突
  2. 对于组件的数据，必须是一个函数，它的值必须是一个返回对象的函数。如果不适用函数返回，那所有的组件实例都共享一个引用，也就是重复使用，这时候如果引用处修改了数据，那么所有数都会修改
  3. prop的定义应该详细，这样即方便后期开发的时候看懂代码，也不会引起Vue报错
  4. 为v-for设置键值，设置键值以后，vue渲染dom的时候就会把dom的变更降到最低
  5. 避免v-if和v-for一起使用，以为vue处理这两个命令的时候，v-for的优先级是高于v-if的，这就导致每次都要渲染完再判断v-if里面的内容，我们可以放在计算属性里面，或者移步到上层
  6. 为组件样式设置作用域，一般现实使用组件样式的时候都会使用scope，或者采用css的BEM策略

    CSS BEM策略

    Block Element Modifier的首字母缩写，是一种css的命名规范，一般写作.block__element–modifier

    B(block)：独立的页面及逻辑单元。 b-name

E(element): 块中的组成部分，不能脱离存在，一般为元素的功能单词或元素的标签。__element

M(modifier): 修饰符，一般为某个元素的样式或处于的一种状态。–modifier

CSS Modules

学这个之前让我想起来，我在使用Vue的时候，有时候样式名字总是带一串字符串，像是哈希字符串，涉及到了CSS Modules。

局部作用域
使用了webpack构建模块的css-loader，把class名称唯一化，使之称为局部的作用域，把css文件输入到style对象中，然后引用style.title代表一个class，构建工具会把类名style.title编译成一个哈希字符串

全局作用域
想要使用全局作用域，不加上哈希字符串，就使用:global(.className)

定制哈希类名
css-loader的默认算法是[hash:base64]，可以再webpck.config.js里定制哈希字符串格式

Vuex状态管理

采用集中式存储管理应用的所有组件的状态，并以相应的规则保证状态以一种可预测的方式发生变化。
有时候很多视图共享一个状态，使用传参非常复杂，通过定义和隔离状态管理中的各种概念并通过强制规则维持视图和状态间的独立性，我们的代码将会变得更结构化且易维护。

State单一状态树
用一个对象就包含了全部的应用层级状态，用以作为唯一数据源，Vuex 通过 store 选项，提供了一种机制将状态从根组件“注入”到每一个子组件中（需调用

当一个组件需要获取多个状态的时候，将这些状态都声明为计算属性会有些重复和冗余。为了解决这个问题，我们可以使用 mapState 辅助函数帮助我们生成计算属性
Getter
Vuex 允许我们在 store 中定义“getter”（可以认为是 store 的计算属性）。就像计算属性一样，getter 的返回值会根据它的依赖被缓存起来，且只有当它的依赖值发生了改变才会被重新计算。
